\qitem{1}{关于 NVIDIA GPU 中流多处理器（Streaming Multiprocessor, SM）的基本结构，下列说法\textbf{正确}的是}{}
\fourchoices{每个 SM 仅包含一个 CUDA Core，所有线程串行执行}{每个 SM 拥有独立的 Warp 调度器，可将 32 个线程组成一个 Warp 进行 SIMT 调度}{SM 内的 Shared Memory 与 L2 Cache 物理上是同一块存储区域}{寄存器文件（Register File）位于 HBM 上，所有 SM 共享访问}

\fourchoices{Register $>$ Shared Memory $>$ L2 $>$ HBM}{HBM $>$ L2 $>$ Shared Memory $>$ Register}{Shared Memory $>$ Register $>$ HBM $>$ L2}{L2 $>$ HBM $>$ Register $>$ Shared Memory}

\qitem{3}{Roofline 模型中，设某算子的算术强度为 $I$（单位：FLOP/Byte），硬件 Ridge Point 为 $I^*=\dfrac{P_{\mathrm{peak}}}{B_w}$。当 $I < I^*$ 时，该算子处于}{}

\qitem{4}{设矩阵乘法 $C_{M\times N}=A_{M\times K}B_{K\times N}$，则完成该 GEMM 所需的浮点运算量（FLOPs）为}{}

\qitem{7}{在 W8A8 量化推理中，设激活 $\mathbf{X}$ 的 per-tensor 缩放因子为 $s_x$，权重 $\mathbf{W}$ 的 per-channel 缩放因子为 $s_w^{(j)}$（$j$ 为输出通道索引）。则等价的 fp32 线性层输出可写为}{}

\qitem{8}{关于 NVIDIA Tensor Core 的 MMA（Matrix Multiply-Accumulate）指令，下列说法\textbf{正确}的是}{}
\fourchoices{Tensor Core 仅支持 FP32 输入与 FP32 累加}{标准 FP16 MMA 指令形状为 $m16n8k16$，即每次处理 $16\times8$ 的输出块，$K$ 维度每次累加 16}{Tensor Core 与 CUDA Core 在硬件上完全独立，无法在同一次 Kernel 中混用}{使用 Tensor Core 的 GEMM 不需要对矩阵做分块（Tiling）}

\qitem{9}{下列关于 GPTQ、AWQ、SmoothQuant 三种量化算法\textbf{实现逻辑}的匹配，\textbf{正确}的是}{}
\fourchoices{GPTQ 基于 Hessian 二阶信息逐列量化权重；AWQ 基于显著权重通道缩放；SmoothQuant 通过 $S$ 矩阵将激活离群点迁移到权重侧}{GPTQ 基于激活值平滑；AWQ 基于 Hessian 逐列量化；SmoothQuant 基于显著通道缩放}{三者均属于 QAT（量化感知训练），需要完整反向传播}{三者均只量化权重，推理时激活值保持 fp32}

\qitem{10}{关于 Triton 与手写 CUDA 的编程模型差异，下列说法\textbf{正确}的是}{}
\fourchoices{Triton 以 Thread 为基本编程单元，开发者需手动管理每个线程的索引}{Triton 以 Block 为基本抽象，编译器自动处理 Thread/Warp 级并行与内存合并访问}{Triton 无法编写自定义 GEMM Kernel，只能调用 cuBLAS}{手写 CUDA 无法做算子融合，Triton 才可以}

\qitem{11}{FlashAttention-2 相比 FlashAttention-1 的主要改进\textbf{不包括}}{}
\fourchoices{减少非矩阵乘法的 FLOPs（如减少 rescale 操作）}{在 Warp 级别更好地并行化 attention 计算}{将 $Q$、$K$、$V$ 的 SRAM 分块策略从单线程块改为跨 Warp 分工}{将注意力计算的 IO 复杂度从 $O(N^2)$ 提升到 $O(N^3)$}

\qitem{12}{关于 XLA / TVM 等深度学习编译器对计算图的优化，下列说法\textbf{正确}的是}{}
\fourchoices{XLA 的 HLO 图优化不支持算子融合，融合只能由 PyTorch eager 模式完成}{TVM 通过 AutoTVM / Ansor 对 Kernel 的 tile size、unroll factor 等超参进行自动搜索}{TVM 只能为 CPU 生成代码，不支持 GPU}{XLA 的 Layout 优化与 Kernel Fusion 无关，二者互不影响}

\begin{multicols}{4}

  \item A

  \item B
  \item D
  \item B
\end{enumerate}

\vspace{0.3cm}
\noindent\textbf{1. B}\quad SM 是 GPU 的基本计算单元，内含多个 CUDA Core、Tensor Core、Warp 调度器和 Shared Memory。(A) 错误：每个 SM 含数十至上百个 CUDA Core。(C) 错误：Shared Memory 是 SM 内片上存储，与 L2 物理独立。(D) 错误：寄存器文件位于 SM 片上。Warp 是 SIMT 调度的基本单位（32 线程）。故选 B。

\noindent\textbf{2. A}\quad 存储层级越靠近计算单元，带宽越高、延迟越低：Register（最快）$>$ Shared Memory $>$ L2 $>$ HBM（最慢、容量最大）。故选 A。

\noindent\textbf{3. B}\quad Roofline 模型：$I<I^*$ 时性能受 HBM 带宽限制（Memory-bound），算力无法跑满；$I>I^*$ 时为 Compute-bound。故选 B。

\noindent\textbf{4. B}\quad 每个 $C_{ij}$ 需 $K$ 次 MAC，共 $MNK$ 次 MAC；1 MAC $=$ 2 FLOPs，故总 FLOPs $=2MNK$。故选 B。

\noindent\textbf{5. B}\quad TPU 采用脉动阵列，权重预加载、数据在阵列中流动完成 MAC；GPU 采用 SIMT 模型，线程可执行不同分支。(A)(C)(D) 均错误。故选 B。

\noindent\textbf{6. C}\quad 融合能减少 HBM 往返和 Launch 开销，但并非任意两算子融合都能减半时间；融合受数据依赖、寄存器压力和算子形状约束。故选 C。

\noindent\textbf{7. B}\quad W8A8 内核流程：$\mathbf{X},\mathbf{W}$ 量化为 INT8 $\to$ INT32 累加 GEMM $\to$ 乘 $s_x\cdot s_w^{(j)}$ 反量化到 fp32。per-channel 权重 scale 是常见做法。故选 B。

\noindent\textbf{8. B}\quad Tensor Core MMA 标准 FP16 形状为 $m16n8k16$；输入 FP16/BF16，累加器 FP32。GEMM 仍需多级 Tiling 才能高效利用 Tensor Core。故选 B。

\noindent\textbf{9. A}\quad GPTQ 用 Hessian 逐列权重量化；AWQ 保护显著权重通道；SmoothQuant 用 $S$ 矩阵平滑激活离群点。三者均为 PTQ。故选 A。

\noindent\textbf{10. B}\quad Triton 以 Block 为编程抽象，编译器负责 Thread/Warp 映射与合并访存；CUDA 需手动管理线程索引。故选 B。

\noindent\textbf{11. D}\quad FlashAttention-2 改进包括减少非 GEMM FLOPs、更好 Warp 并行、优化 SRAM 分块；IO 复杂度仍为 $O(N^2d^2/M_{\mathrm{sram}})$，不会升到 $O(N^3)$。故选 D。

\noindent\textbf{12. B}\quad TVM/Ansor 自动搜索 tile size、unroll 等；XLA 支持 GPU 且含 Fusion、CSE、Layout 等 Pass。故选 B。

\subsection*{\heiti 三、解答题答案}

\subsubsection*{19. Roofline 定量分析（10 分）}

\noindent\textbf{(1) 算术强度（3 分）}

FLOPs $=2b\cdot h\cdot 4h=8bh^2$。HBM 访问量（fp16，2 字节/元素）：读 $A$ 为 $2bh$ 元素，读 $B$ 为 $2\cdot h\cdot 4h=8h^2$ 元素，写 $C$ 为 $2b\cdot 4h$ 元素，总字节 $=2(2bh+8h^2+8bh)=4bh+16h^2+16bh=4(5bh+4h^2)$。故
\[
I=\frac{8bh^2}{4(5bh+4h^2)}=\frac{2bh^2}{5bh+4h^2}.
\]

\noindent\textbf{(2) 瓶颈判断（3 分）}

$b=1$，$h=4096$：$I=\dfrac{2\times4096^2}{5\times4096+4\times4096^2}\approx\dfrac{2\times4096}{4\times4096+5}\approx0.49$ FLOP/Byte。$I^*\approx153$ FLOP/Byte，$I\ll I^*$，该 GEMM 处于 \textbf{Memory-bound}。

\noindent\textbf{(3) 优化策略（4 分）}

\begin{itemize}[leftmargin=2em,itemsep=0pt,parsep=0pt,topsep=0pt]
  \item \textbf{增大 Batch Size $b$}：$I\propto b$（分子 $8bh^2$，分母中 $bh$ 项主导时），提高算术强度，向 Compute-bound 靠近。
  \item \textbf{算子融合}：将 LayerNorm/RMSNorm 与后续 Linear 融合，减少中间激活写回 HBM，降低有效字节数。
  \item \textbf{分块 GEMM + Tensor Core}：虽不能改变 Roofline 定性结论，但提高实际带宽利用率，使 achieved FLOPs/s 更接近 Roofline 上限。
\end{itemize}

\subsubsection*{20. GEMM 三级分块（10 分）}

\noindent\textbf{(1) 三级分块（4 分）}

\begin{itemize}[leftmargin=2em,itemsep=0pt,parsep=0pt,topsep=0pt]
  \item \textbf{Thread Block Tile}：驻留 Shared Memory；每个 Block 负责 $C$ 的一个 $B_M\times B_N$ 子块，协作加载 $A,B$ 的 $B_M\times B_K$、$B_K\times B_N$ 片。
  \item \textbf{Warp Tile}：Warp 内分工，每个 Warp 处理 Block Tile 的子区域，协调 Shared Memory 访问。
  \item \textbf{Register Fragment}：驻留寄存器；每个线程持有 $A,B$ 的小块（Fragment），通过 Tensor Core MMA 累加。
\end{itemize}

\noindent\textbf{(2) HBM 访问量（4 分）}

分块后 $A$ 每元素被复用约 $B_N/B_K$ 次（沿 $N$ 方向），$B$ 每元素被复用约 $B_M/B_K$ 次。总 HBM 读取量量级为 $O\!\left(\dfrac{MNK}{B_K}\right)$。相比朴素 $O(MNK)$ 无复用，降低约 $\dfrac{B_M B_N}{B_K}$ 倍（典型 $B_M=B_N=B_K=128$ 时约 128 倍量级）。

\noindent\textbf{(3) Coalesced Access 与 Bank Conflict（2 分）}

Coalesced Access：相邻线程访问连续 Global Memory 地址，合并为少量 Cache Line 事务。Bank Conflict：多个线程同时访问 Shared Memory 同一 Bank 不同地址导致串行化；规避方法包括 padding（如 $B_K+1$ 列）使访问映射到不同 Bank。

\subsubsection*{21. FlashAttention 内核原理（10 分）}

\noindent\textbf{(1) 标准注意力 HBM 读写（3 分）}

$S$：写 HBM 1 次（$N\times N$）；$P=\mathrm{softmax}(S)$：读 $S$ 1 次、写 $P$ 1 次；$O=PV$：读 $P$ 1 次。$S,P$ 合计：写 2 次、读 2 次（仅计 $S,P$ 矩阵）。

\noindent\textbf{(2) Online Softmax 更新（4 分）}

设新块得分 $S^{(j)}$，旧 running max $m$，旧 sum $\ell$，旧输出 $O$。更新：
\[
m'=\max(m,\max(S^{(j)})),\quad
\ell'=\ell\cdot e^{m-m'}+\sum e^{S^{(j)}-m'},\quad
O'=\dfrac{\ell}{\ell'}O\cdot e^{m-m'}+\dfrac{e^{S^{(j)}-m'}}{\ell'}V^{(j)}.
\]

\noindent\textbf{(3) IO 复杂度与 Recomputation（3 分）}

分块大小受 $M_{\mathrm{sram}}$ 约束，每块 $Q_i,K_j,V_j$ 驻留 SRAM，不写出完整 $S,P$。HBM IO 复杂度 $O(N^2d^2/M_{\mathrm{sram}})$。反向传播不保存 $S,P$，在反向时重算前向分块（Recomputation），以额外计算换取 HBM 读写减少。

\subsubsection*{22. 量化内核实现（10 分）}

\noindent\textbf{(1) SmoothQuant 公式（3 分）}

\[
s_j=\frac{c_j^\alpha}{w_j^{1-\alpha}},\quad \alpha\in(0,1].
\]
$\alpha=0.5$ 表示激活与权重各承担一半量化难度，离群点均匀迁移。

\noindent\textbf{(2) INT4 Packed GEMM（4 分）}

4 个 INT4 打包进 1 个 INT32。Kernel 加载 INT32 后通过位操作（shift/mask）解包为 INT4，符号扩展后送入 INT32 累加器做 MAC；累加完成后乘 scale 反量化到 fp16/fp32。权重量化后显存降为 1/4。

\noindent\textbf{(3) GPTQ Hessian 更新（3 分）}

GPTQ 逐列量化权重，用 $H=2X^{\top}X$（Hessian）衡量各权重对损失的影响。量化第 $j$ 列产生误差 $\delta_j$，按 $H^{-1}$ 更新未量化列：$w_i\leftarrow w_i-\dfrac{H^{-1}_{ij}}{H^{-1}_{jj}}\delta_j$，将误差传播到剩余列以最小化 $\|H(\hat{w}-w)\|^2$。

\subsubsection*{23. 编译器与融合（10 分）}

\noindent\textbf{(1) XLA HLO Pass（3 分）}

\begin{itemize}[leftmargin=2em,itemsep=0pt,parsep=0pt,topsep=0pt]
  \item \textbf{Kernel Fusion}：合并相邻算子，减少中间张量写回。
  \item \textbf{CSE（公共子表达式消除）}：合并重复计算。
  \item \textbf{Layout Optimization}：选择 NCHW/NHWC 等内存布局以匹配高效 Kernel。
\end{itemize}

\noindent\textbf{(2) RMSNorm + Linear 融合（4 分）}

未融合：输入 $\to$ RMSNorm $\to$ $\mathbf{h}$ 写 HBM $\to$ Linear 读 $\mathbf{h}$ $\to$ 输出写 HBM。融合：输入 $\to$ RMSNorm（片上）$\to$ Linear（片上）$\to$ 输出写 HBM。节省 $\mathbf{h}$ 的 1 次写 + 1 次读 $=2\times d\times 2$ 字节（fp16）。

\noindent\textbf{(3) Triton autotune（3 分）}

\texttt{@triton.autotune} 对 \texttt{BLOCK\_SIZE}、\texttt{num\_warps} 等配置枚举，每种配置实际运行 benchmark，选耗时最短者。代价模型关注：寄存器占用（影响 Occupancy）、Shared Memory 用量、 achieved FLOPs/s 或 latency。

\subsubsection*{24. 综合性能诊断（10 分）}

\noindent\textbf{(1) Roofline 与 KV Cache（3 分）}

\texttt{dram\_throughput} 高、\texttt{sm\_\_throughput} 低 $\Rightarrow$ Memory-bound。Decode 每步 FLOPs $O(d)$ 但需读全量 KV Cache。单请求 KV 读取字节：
\[
M_{\mathrm{KV}}=2\times L\times n\times h\times d_h\times\text{bytes\_per\_scalar}
\]
（$L$ 层，$n$ 当前长度，$h$ 头数，$d_h$ 每头维度，因子 2 为 K+V。）

\noindent\textbf{(2) 优化栈（4 分）}

\begin{itemize}[leftmargin=2em,itemsep=0pt,parsep=0pt,topsep=0pt]
  \item \textbf{W8A8}：权重/激活 INT8，HBM 读取量约减半，缓解带宽瓶颈。
  \item \textbf{Kernel Fusion}：RMSNorm+Linear、SiLU+Mul 等融合，减少中间写回，提高算术强度。
  \item \textbf{Custom Attention}：FlashAttention-2 / PagedAttention，分块 attention、减少 KV 碎片，提高带宽利用率与 SM 占用。
  \item \textbf{增大并发 Batch}：提高 GEMM $M$ 维，使线性层从 Memory-bound 转向 Compute-bound，提升 SM 利用率。
\end{itemize}

\noindent\textbf{(3) Nsight 指标（3 分）}

\begin{itemize}[leftmargin=2em,itemsep=0pt,parsep=0pt,topsep=0pt]
  \item \textbf{sm\_\_throughput}：优化后应上升，说明计算单元更忙。
  \item \textbf{dram\_\_throughput / dram\_bytes\_read}：W8A8 和 Fusion 后应下降或单位 token 降低。
  \item \textbf{achieved\_occupancy}：分块和寄存器优化后应提高；过低说明 Warp 空闲或寄存器压力过大。
\end{itemize}
